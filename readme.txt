     =======电子词典======

一、功能说明：
    1、用户可以登录和注册
        登录凭借用户名和密码登录
        注册要求用户补习填写用户密码，其他内容自己拟定
        用户名要求不重复

    2、用户信息必须长期保存，以满足随时登录
        要求用数据库保存

    3、能够满足多个用户同时登录操作的需求

    4、项目分为客户端和服务端，客户端负责请求和展示数据，其余处理由服务端完成

    5、客户端启动即进入一级界面(界面print打印)，一级几面功能
        登录 注册  退出

    6、用户登陆后即进入二级界面，(注册后是否处于登录状态自定)，二级界面功能
        查单词  查看历史记录 注销
        查单词：可以循环输入单词，获取单词解释，
            提示：每个单词占一行
                 单词按照顺序排列
                 单词和解释之间有空格

        查看历史记录： 
            格式：   name    world  time  
            可以显示登录人员查的所有单词 或者最近10条
        注销：返回一级界面 

二、步骤
    1、确定技术点
        1、用什么并发    fork多进程
        2、用什么套接字  tcp套接字
        3、用什么数据库  MySQL
        4、文件处理还是数据库查询   文件处理

    2、建立数据库，设计表
        用户表：    id  name  passwd  
             create table user(id int primary key auto_increment, 
             name varchar(32) not null,pass varchar(16) default '0000');

        历史记录：  id  name  word  time
            create table hist(id int primary key auto_increment,name 
            varchar(32) not null,word varchar(32) not null,time varchar(64));

        单词表：    id  word  interpret 
             create table words(id int auto_increment primary key,word 
             varchar(32),interpret text);  

    3、结构设计，仿照ftp文件服务器
        服务器： 创建套接字  --  创建父子进程-- 子进程等待客户端请求 -- 父进程继续等待连接下一个客户端
                具体请求调用具体函数与处理
        客户端：创建套接字 -- 发起连接请求 -- 进入一级界面 -- 发起请求（登录、注册、退出）
               登录成功进入二级界面 -- 请求（查询，历史记录）


    4、完成通信的搭建
        完成
    5、功能分析，根据分析完成功能代码
        1、登录：
            客户端：
                1、输入用户密码
                2、将信息发送给服务端
                3、得到服务器反馈
                4、进入二级界面
            服务端：
                1、接收请求
                2、判断是否运行登录
                3、反馈结果
            

        2、注册：
            客户端：
                1、输入注册信息
                2、将信息发送给服务器
                3、得到服务器反馈
            服务端：
                1、接收请求
                2、判断是否允许你注册
                3、将结果反馈给客户端
                4、将注册信息插入数据库

        3、查单词
            客户端：
                1、发送请求  Q name word
                2、接收返回结果
                3、## 退出查词  
            服务端：
                1、接收请求
                2、查找单词，反馈结果
                3、插入历史记录
                
        4、查询历史记录
            客户端：发送要查询的单词，接收查询结果并展示结果
            服务端：接收查询的单词并将数据库的查询就结构返回给客户端
            数据库端；查询单词
            * 循环多次查询单词
